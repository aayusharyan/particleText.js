ParticleText.js supports responsive breakpoint-based configurations. This allows you to adjust particle size, explosion radius, and other parameters based on screen size. The library uses 9 breakpoints from mobile (xxxs) to 4K displays (xxxl).

Configuration:

initParticleJS('#canvas', { text: 'RESPONSIVE', colors: ['#695aa6', '#ABC121'], // Responsive particle sizes particleRadius: { 'xxxs': { base: 1, rand: 0.5 }, // Tiny for small screens 'xs': { base: 1.5, rand: 1 }, // Small for phones 'md': { base: 2, rand: 1 }, // Medium for tablets 'lg': { base: 3, rand: 2 }, // Larger for desktops 'xxxl': { base: 4, rand: 2 } // Largest for 4K displays }, // Responsive explosion radius explosionRadius: { 'xxxs': 15, // Small touch zone for tiny screens 'xs': 20, // Phone-friendly 'md': 50, // Tablet size 'lg': 100, // Desktop with mouse 'xxxl': 200 // Large displays } });

üí° Best Practices:

  • Breakpoint Inheritance: You don't need to specify all breakpoints - larger screens inherit from smaller ones
  • Mobile First: Start with 'xs' or 'sm' configuration and override for larger screens
  • Performance: Use smaller particles and lower explosion radius on mobile to maintain smooth FPS
  • Touch vs Mouse: Smaller explosion radius works better for touch interfaces
