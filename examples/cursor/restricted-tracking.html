Restricted Cursor Tracking

Particles only react when cursor is inside the canvas

Configuration Used:

  • trackCursorOnlyInsideCanvas: true
  • Particles will ONLY explode when your cursor is inside the canvas boundaries
  • When cursor moves outside, particles immediately return to their original positions
Try it: Move your cursor outside the canvas boundaries and watch the particles snap back to their positions instantly!

When to Use This Setting?

Enable trackCursorOnlyInsideCanvas: true when:

  • Multiple canvas elements: Prevent interference between adjacent particle effects
  • Page with interactive elements: Users might interact with buttons/links near the canvas
  • Defined interaction zones: You want clear boundaries for where the effect is active
  • Performance concerns: Reduce unnecessary particle calculations when cursor is far away
  • UI/UX clarity: Make it explicit that the effect only works within the canvas area

Keep default (false) when:

  • Smooth experience: You want seamless transitions as cursor approaches the canvas
  • Single large canvas: The canvas takes up most of the viewport
  • Creative effects: Particles reacting before cursor enters creates anticipation
